# セッション内容を技術記事化

このコマンドは、**今回の作業セッション**を対外向けの技術記事として整理する。
センシティブ情報は必ず除外し、再現可能な内容にする。

## 1. 情報収集

以下を収集して、記事の材料にする。

```bash
ROOT="$(git rev-parse --show-toplevel)"
DATE="$(date +%Y-%m-%d)"
OUT_DIR="$ROOT/docs/articles"
OUT_FILE="$OUT_DIR/${DATE}_session-article.md"

git status -sb
git log -5 --oneline
git diff --stat
```

必要に応じて以下も確認：
- 参照したドキュメント（`docs/` など）
- 変更したコマンド/設定（`/commands`, `/rules`, `.claude`, `.gemini` 等）
- 実行したテストや検証コマンド

## 2. 記事テンプレート作成

```bash
mkdir -p "$OUT_DIR"
cat <<'EOF' > "$OUT_FILE"
# タイトル（具体的に）

## 背景 / 課題
- なぜ必要だったか
- どんな問題があったか

## 取り組んだこと
- 実施内容の要点
- 重要な判断や方針

## 実装・設定のポイント
- 具体的な設定や構成（必要な範囲で）
- 重複回避や運用上の工夫

## 成果
- 何が改善されたか
- 何が明確になったか

## 学び / 次のアクション
- 得られた知見
- 今後の課題
EOF
```

## 3. 内容の埋め込み

- 収集した情報をもとに、**対外向けに分かる言葉**で本文を埋める。
- 社外秘/個人情報/未公開URLなどは省く。
- 箇条書き中心で簡潔に。必要なら図/リンクを追記。

## 4. 出力
obsidianのINBOXに出力すること。
`$OUT_FILE` をユーザーに提示すること。
