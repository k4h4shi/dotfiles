# 状況共有（親Issueコメント）

並行開発では「途中で出てきた重要な情報」が散逸すると、他エージェントがすぐビルドトラップに落ちる。
共有すべきコンテキストは **スプリント親Issue** に集約し、随時コメントで状況を更新する。

## いつ使うか
- 何かが完了した（PR作成/マージ、仕様確定、回帰止血）
- 追加の観点が出てきた（仕様根拠、衝突、スコープ変更）
- ブロッカーが出た（CI、依存、環境）
- 次の一手を示したい（他エージェントに取ってほしい作業）

## コメントテンプレ（最小）
```bash
gh issue comment <PARENT_ISSUE_NUMBER> --body \"$(cat <<'EOF'
## Status YYYY-MM-DD HH:MM
- Done:
  - #190 ... (PR #...)
- New context:
  - <仕様根拠/決定/追加観点>
- Blockers:
  - <詰まり/リスク>
- Next:
  - <次の一手（担当してほしいIssue/作業）>
EOF
)\"
```
